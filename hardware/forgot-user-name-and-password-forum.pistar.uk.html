KE7FNS wrote: ↑ Thu Dec 05, 2019 6:45 pm Theres tons of info out there on the web.

https://www.raspberrypi.org/forums/view... hp?t=20397

The method detailed in that thread doesn't work anymore. Grub and boot process has changed so keyboard doesn't work anymore with this method. So when you boot with init=/bin/sh in the cmdline.txt file on any of the modern kernels you end up with a prompt but no keyboard as the kernel HID modules haven't loaded yet. Only works on old kernels/grub versions. It might still work on the the older pi-star 3 images if someone hasn't updated them to latest kernel updates but it doesn't work on any of the new 4.1 images which have the newer kernel by default.

The systemd.unit=emergency.target also doesn't work on the new version of either as the root account is locked by default in the image and emergency.target requires root account to be unlocked.

So the easier methods of password recovery no longer work. That means for account password recovery need to be able to put the card in a system running linux (installed or booted from a CD or USB Stick). Then can mount the volume to edit the passwd and shadow files to reset the password.
