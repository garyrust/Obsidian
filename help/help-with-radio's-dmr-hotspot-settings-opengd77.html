If rebooting the hotspot fixes it, then whatever is causing problem is not in the radio

You could try using using PiStar rather than WPSD

In your when using a Duplex hotspot as simplex... In PiStar, AFIK, you need to change 2 settings

1. Select the radio button at the top of the screen to select Simplex
2. Select the modem type as Simplex

This will then only use one of the transceivers on the duplex modem

I run duplex hotspots in simplex mode, but they all use Rpi3B+ boards, because I find the PiZero quite slow, albeit I had PiZero V1 not the V2 which I presume is faster

I'd also recommend not connecting to multiple networks at the same time. i.e you say you are connecting to BM and TGIF, which will mean that behind the scenes you are running DMR Gateway, which isn't the easiest thing to configure.

WPSD must be configuring DMR Gateway, but potentially there is some miss configuation going on

In my experience most people end up running multiple hotspots, one per network rather than attempting to route multiple networks through one simplex hotspot.

That being said, I guess using a duplex hotspot with one network per TS, then you could run 2 networks without major clashes, but on a simplex hotspot, I think things would likely end up being problematic
