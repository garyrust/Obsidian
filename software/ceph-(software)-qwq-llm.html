To use your existing CEPH installation on MX 23 Linux with a GUI, you can leverage Ceph Dashboard, an official web-based interface included in modern Ceph versions. Below is a step-by-step tutorial to set it up and use it:

Step 1: Ensure Ceph and Dependencies Are Installed

First, confirm that your Ceph cluster is running properly:

sudo ceph status

If you see output like health: HEALTH_OK, proceed. If not, fix any issues first.

Install missing dependencies for the dashboard (if needed):

sudo apt update
sudo apt install python3-pygments  # Required for markdown rendering in the dashboard

Step 2: Enable the Ceph Dashboard

  1. Enable the dashboard module on your Ceph Manager (mgr):

    sudo ceph mgr module enable dashboard

  2. Set a default username and password for accessing the dashboard:

    # Generate an admin user (replace 'admin' with your desired username)
sudo ceph dashboard create-user admin -i

# Enter a password when prompted.

    If you don’t set this, the dashboard will use a random token accessible via ceph mgr services.

Step 3: Configure HTTPS Access

The Ceph Dashboard uses HTTPS by default. To avoid browser warnings:

  1. Generate SSL certificates (optional but recommended):

    sudo ceph dashboard setup-https --enable

    This creates self-signed certs in /etc/ceph/ceph-dashboard/.

  2. Alternatively, use a self-signed certificate temporarily:
    The dashboard will work without this, but you’ll see security warnings.

Step 4: Access the Dashboard

  1. Open your browser (e.g., Firefox/Iceweasel) and navigate to:

    https://<Ceph-MGR-Node-IP>:8443

    Replace <Ceph-MGR-Node-IP> with the IP of the node where Ceph Manager is running.

  2. Accept the self-signed certificate (if you didn’t generate custom certs).

  3. Log in using:

    • Username: admin (or the user you created earlier).
    • Password: The one you set during step 2.

Step 5: Basic Ceph Dashboard Usage

Once logged in, here’s what you can do:

1. Monitor Cluster Health

  • Navigate to Monitoring > Overview to see cluster health, storage usage, and alerts.

2. Manage Pools & OSDs

  • Go to Storage > Pools to create/modify pools.
  • Under Storage > OSDs, view disk status or replace failed drives.

3. User Management

  • Create/administer users under Access Control > Users.

Step 6: Troubleshooting Common Issues

  • Dashboard not accessible?

    • Check if the dashboard module is enabled: 
      sudo ceph mgr module ls | grep dashboard
    • Ensure port 8443 isn’t blocked by your firewall. 
      sudo ufw allow 8443/tcp

  • Login Credentials Forgotten?
    Reset the admin password:

    sudo ceph dashboard ac-user-password-reset admin

Alternative GUI Tools (Optional)

If you prefer a desktop app, consider these unofficial tools:

  1. CephGUI: A lightweight Python-based tool.
    • Install via PPA or from source: GitHub Repo.
  2. Rook Dashboard: If using Rook (Kubernetes-based), but this requires Kubernetes setup.

Next Steps

  • Explore the Ceph Dashboard documentation: Ceph Docs
  • Use CLI commands alongside the GUI to deepen your understanding: 
    sudo ceph osd df    # Show OSD usage
sudo ceph df        # Display cluster space

This setup provides a user-friendly way to manage Ceph while learning its core concepts. Let me know if you hit any snags!
